## Day 009 — Lesson A (Foundations)

### Objectives

- Do a light **review + consolidation** of Week 1 foundations.
- Mix arrays, strings, objects, conditionals, and simple tests.
- Practice thinking about problem decomposition on your own.

### Concepts for today

- Combining earlier ideas:
  - array scanning,
  - filtering and mapping,
  - frequency-style summaries,
  - tiny tests/assertions.
- Thinking in terms of:
  - “input shape”,
  - “output shape”,
  - and “steps to get from one to the other”.

### Reading + examples

You will design a handful of small helpers around a **generic list of items**
and write basic checks for them. For example:

- A helper that filters items based on a condition.
- A helper that derives a summary object (counts, totals).

Questions:

- How would you break a slightly vague requirement into concrete helpers?
- What do you test first?
- How do you avoid mixing multiple concerns in one function?

### What you’ll implement in the exercise

In the exercise file you will:

- Implement 3–4 small helpers that each use **different earlier skills**.
- Add at least one simple assertion or console-based test per helper.
- Treat this as a “checkpoint”: can you drive the code more independently now?\*\*\*
